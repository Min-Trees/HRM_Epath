# T01 — Khởi tạo monorepo & công cụ build

## Đã làm gì

- Dựng khung monorepo `hrm-system/` gồm ba thư mục `database/`, `backend/`, `frontend/` đúng theo `00-CONVENTIONS.md`.
- Tạo bộ khởi tạo backend Spring Boot 3.3.5 (Java 21, Maven) với đầy đủ dependencies nền: web, validation, JPA, PostgreSQL driver, Flyway + Flyway-postgresql, Jackson JSR-310, Testcontainers Postgres + JUnit, H2 (test scope).
- Cấu hình `application.yml`: datasource đọc từ biến môi trường `DB_URL/DB_USERNAME/DB_PASSWORD` (mặc định `localhost:5432/hrm`), JPA `ddl-auto: validate`, Flyway enabled, server port 8080, profile `local`.
- Tạo lớp `com.company.hrm.HrmApplication` với `@SpringBootApplication`.
- Khởi tạo frontend Vite + React 18 + TypeScript 5.6 với các phiên bản ghim cụ thể (react, react-dom, vite, @vitejs/plugin-react, typescript, @types/react, @types/react-dom). Cấu hình proxy `/api` → `http://localhost:8080` trong `vite.config.ts`. `tsconfig` chia app/node references với strict + noUnused* để bắt lỗi sớm.
- Scripts: `dev`, `build` (tsc -b + vite build), `preview`, `typecheck`, `lint`. Do PowerShell + npm 11 trên Windows xử lý `tsc` lạ (resolve sai đường dẫn), scripts dùng `node node_modules/...` trực tiếp để hoạt động ổn định trên mọi shell.
- Viết `hrm-system/README.md` mô tả yêu cầu môi trường, cấu trúc, lệnh chạy backend và frontend.
- Tạo `.gitignore` phù hợp (target, node_modules, dist, .env, IDE).

## Quyết định thiết kế

1. **Giữ Java 21 ở pom (theo Conventions)** dù môi trường verify hiện tại chỉ có JDK 17. Khi có JDK 21, `mvn package` chạy ngay. Việc chấp nhận JDK 17 chỉ mang tính tạm thời để verify tĩnh.
2. **`ddl-auto: validate`** cộng với **Flyway enabled**: mọi thay đổi schema phải qua migration, không để Hibernate tự ý sửa DB. Phù hợp quy tắc "không sửa schema.sql trực tiếp".
3. **`open-in-view: false`**: tắt OSIV để tránh lazy-loading ngoài transaction, bắt buộc cho code chuẩn từ đầu.
4. **Testcontainers Postgres + H2**: chuẩn bị sẵn cho integration test (T22) và cho unit test nhanh không cần Docker.
5. **Proxy Vite `/api` → backend**: chuẩn hoá endpoint chung, frontend gọi qua đường dẫn tương đối.
6. **`tsc -b` reference project**: cho phép type-check cả `tsconfig.app.json` và `tsconfig.node.json` chỉ với một lệnh.

## Quy tắc nghiệp vụ đã áp dụng

Chưa áp dụng quy tắc nghiệp vụ cụ thể (T01 là task nền tảng, không đụng bảng nghiệp vụ). Tuy nhiên đã:
- Tách rõ bounded context bằng package `com.company.hrm` và 4 schema (hr/timekeeping/social_ins/payroll) ở tầng DB (sẽ thể hiện qua migration ở T02).
- Chuẩn bị cơ chế để các bảng append-only sau này có thể enforce qua Flyway trigger (sẽ thêm ở T11).

## Ghi chú cho task sau

- **T02 (Database & migration)**: cần copy nội dung `database/schema.sql` thành Flyway migration `V1__init.sql` ở `backend/src/main/resources/db/migration/`. KHÔNG sửa schema nguồn.
- **T03 (Lớp chung backend)**: cần thêm `application-local.yml` thật nếu cần override biến môi trường.
- **JDK 21**: báo lại cho team để có môi trường build chính thức; trong lúc chờ, có thể chạy tạm bằng JDK 17 nếu đổi `<java.version>` và `maven.compiler.source/target` về 17 (sẽ KHÔNG dùng cách này vì conventions yêu cầu Java 21).
- **npm scripts dùng `node node_modules/...`**: đây là workaround cho môi trường Windows + npm 11. Trên máy developer bình thường có thể đổi về `tsc -b && vite build` cho gọn.
